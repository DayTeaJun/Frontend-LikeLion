한 명당 한 페이지의 작업은 필수입니다.
로그인
(메인 화면, 이메일 로그인 화면)
-> 서버에 POST
-> 로그인 성공시 프로필 정보 불러오기 서버에 GET

회원가입
(프로필 사진, 사용자 이름, 계정ID, 소개글)
-> 서버에 POST
-> 중복된 이메일 검증 서버에 POST
-> 계정 검증 서버에 POST

splash가 어디로 들어가는가? -> 홈화면, 로그인화면
홈화면 및 로그인화면 판단
-> 로그인 했을 때 서버에서 불러온 내용을 로컬스토리지 안에 토큰이 저장되어있는지 없는지 판단후, 토큰이 있다면 홈화면, 없다면 로그인 화면
(토큰 검증)
-> 서버에 GET

홈 화면
(내것을 제외한, 팔로우된 사용자들이 올린 게시글을 보여줘야 함)
-> 서버에 팔로우된 사용자들이 올린 게시글 GET

검색
(사용자 이름을 검색 및 누르면 프로필로 이동)
-> 서버에 사용자 목록을 GET

사용자 프로필 페이지 ( V )
(사용자 이름, 계정ID, 소개글, 팔로워 및 팔로잉 수, 판매 상품)
-> 서버에서 해당 사용자 정보 불러옴
(내 게시글 목록)
-> 서버에 GET
(팔로우 버튼)
-> 서버에 POST
(언팔로우 버튼)
-> 서버에 DELETE

팔로워, 팔로잉 목록 ( V )
(팔로워 버튼 -> 팔로워 한사람들 목록 페이지, 팔로잉 버튼 -> 팔로잉 해준사람 목록 페이지)
-> 서버에 GET

내 프로필 수정 ( V )
(사용자 이름, 계정ID, 소개글)
-> 서버에 PUT

상품 등록 ( V )
(상품 이미지, 상품명, 가격, 판매링크 -> 저장 클릭시, 상품이 등록됨)
-> 서버에 POST
(상품 리스트)
-> 서버에 GET
(상품 상세)
-> 서버에 GET

상품 클릭시 모달 ( V )
(상품 수정)
-> 서버에 PUT
(상품 삭제)
-> 서버에 DELETE


게시글 작성 페이지
(하단 메뉴바 게시글 작성, 글 입력 및 사진 업로드)
-> 서버에 POST

게시글 상세 페이지
-> 서버에 GET

게시글 댓글 페이지
(댓글 작성)
-> 서버에 POST
(댓글 리스트)
-> 서버에 GET
(댓글 삭제)
-> 서버에 DELETE
(댓글 신고)
-> 서버에 POST



채팅 목록
(현재 대화가 진행중인 채팅 목록)
-> 서버에 GET

채팅방
(채팅 내역 및 채팅 입력창, 채팅방 나가기 모달)
-> 도전과제도 미구현

하단 탭 메뉴
(홈, 채팅, 게시물 작성, 프로필)
* 모든 페이지는 하단 탭 메뉴가 고정되어 활성화 되있음

좋아요 버튼
* 게시글이 있는 곳 모든 페이지
(좋아요)
-> 서버에 POST
(좋아요 취소)
-> 서버에 DELETE

모달 버튼
(헤더 -> 설정 및 개인정보, 로그아웃)
-> 토근 정보 및 토큰 삭제
(내가 작성한 게시글 ->
(수정)
-> 서버에 PUT
(삭제)
-> 서버에 DELETE)
(다른 사용자가 작성한 게시글 -> 신고)
-> 서버에 POST
